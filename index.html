AI 보이스피싱 탐지 — FP Rate 분석 기반 제품화 판단

2025.09 — 2025.12
전체/모델 개발 리드 (3인 팀) · 라온시큐어

보이스피싱 통화를 AI로 실시간 탐지하는 연구. Recall 99.34% 달성했으나, 높은 Recall 자체가 과적합 시그널이었고 실서비스 품질에 미달하여 제품화를 보류했다.

판단 근거: Recall 99.34%는 model robustness 관점에서 과적합으로 판단. FP Rate 22%(OOD 샘플에 대한 robustness 부족 — 정상 통화 약 5건 중 1건 오탐지). 실제 사용 시 체감 오탐률이 수치보다 더 높아 사용자 신뢰를 훼손할 수준으로 분석. 출시를 보류하고 후속 개선 전략(Hard Negative Mining, 정상 대화 합성 데이터 보강)을 수립.
  • KoBERT 기반 모델 아키텍처 설계 및 학습 파이프라인 구축.
  • "Recall이 높으면 출시"가 아닌, 과적합 여부와 체감 품질까지 검증하여 Go/No-Go를 판단한 경험.

AI 딥페이크 탐지 — 온디바이스 vs 클라우드 아키텍처 전환 설계

2024.04 — 2024.12
앱 개발 리드 / 모델 개발 서브 (6인 팀) · 라온시큐어

오픈소스 딥페이크 탐지 모델을 모바일 보안 서비스에 응용. 모델 성능·추론 속도·UX 트레이드오프를 분석해 아키텍처 전환을 설계했다.

설계 판단: 기존 모델의 아시아권 탐지 한계(distribution shift)를 식별, 자체 데이터셋 구축(domain adaptation)으로 정확도 30%p 개선(95% 달성, 자체 테스트셋 기준). 모델 양자화(FP32→INT8) 후 온디바이스 추론 속도 및 MediaProjection 캡처 해상도 제약(최대 720p)을 측정 — 실시간 영상통화(30fps) 대비 처리율 부족으로 온디바이스 실시간 탐지 불가 판단. 정량적 근거를 기반으로 클라우드 비동기 아키텍처로 전환 설계하여 서비스 출시 일정 준수.
  • 서비스 흐름: 사용자가 영상 업로드 또는 URL(유튜브/SNS) 입력 → 클라우드 탐지 → 딥페이크 확률(%) 결과 제공.
  • 50만 다운로드 B2C 앱(라온 모바일 시큐리티)에 탑재하여 서비스 출시.
